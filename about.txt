Гипервизор - программа или аппаратная схема, обеспечивающая или позволяющая одновременное, параллельное выполнение нескольких операционных систем на одном и том же хост-компьютере.

Виртуальная машина - host OS + hypervisor + виртуальная машина со своей OS.
Накладные расходы.

Контейнер - host OS + Docker + контрейнер.
Использует host OS, не создает других OS.
В контейнере нет OS, только приложения, библиотеки и зависимости.

Контейнерный движок - Docker.
Docker host      - компьютер, на котором работает Docker.
Docker daemon    - фоновый процесс.
Docker client    - терминал, API, графический интерфейс.
Docker image     - неизменяемый образ, из которого разворачивается контейнер.
Docker container - развернутое и запущенное приложение.
Dockerfile       - инструкция для создания образа.
Docker registry  - репозиторий для хранения образов. Docker Hub.
Docker compose   - разворачивать и настраивать несколько контейнеров одновременно.
Docker Swarm     - оркестрация, создание и управление кластерами хостов. Много экземпляров одного контейнера на разных машинах.
Kubernetes       - оркестрация, как Docker Swarm, но мощнее.

WSL - Windows Subsystem for Linux — подсистема ОС Windows 10, позволяющая разработчикам, тестировщикам запускать нативные приложения Linux, писать скрипты, выполнять команды непосредственно из Windows.


Dockerfile
FROM: родительский образ
LABEL: метаданные
MAINTAINER: об авторе
ENV: переменные окружения
RUN: запускает команды
COPY: копировать файлы/директории в контейнер
ADD: как COPY + распоковывает архивы
CMD: команда и аргументы для выполнения внутри контейнера
WORKDIR: установить рабочую директорию
ARG: переменная для передачи Docker'у во время сборки
ENTRYPOINT: как CMD
EXPOSE: открыть порт контейнеру
VOLUME: создает точку подключения директории для добавления и хранения постоянных данных
